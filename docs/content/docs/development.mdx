# Development

## Prerequisites

- **Rust** 1.93+ (with `cargo`)
- **Node.js** 22+ (with `pnpm`)
- **Docker** and Docker Compose
- **PostgreSQL** 17 (via Docker or local install)

## Quick Start

```bash
# Clone the repository
git clone <repo-url> && cd sharkie

# Copy environment files
cp sharkie.env.example sharkie.env          # production/docker defaults
cp sharkie_db.env.example sharkie_db.env    # postgres credentials
cp sharkie.local.env.example sharkie.local.env  # local dev overrides
# Edit sharkie.env with your CURRENCY_API_KEY

# Start everything (db in Docker with auto-restore, backend + frontend natively)
make dev
```

`make dev` will:
1. Start PostgreSQL in Docker
2. Auto-restore the latest dump from `./dumps/` (skips if DB already has data)
3. Start the Rust backend on port 3000
4. Start the Next.js frontend on port 3001

## Environment Files

The project uses **three env files**. All are gitignored (`*.env`); only `*.env.example` templates are committed.

The backend loads env files in order: `sharkie.local.env` first, then `sharkie.env`. Since dotenvy doesn't overwrite, **the first loaded value wins** — local overrides take precedence.

### `sharkie.env` — Shared defaults (Docker Compose + fallback)

Used by the `app` service in `docker-compose.yml` via `env_file:`. Also loaded by `cargo run` as fallback for any vars not set in `sharkie.local.env`.

```bash
# sharkie.env.example — copy to sharkie.env

DATABASE_URL=postgresql://sharkie:sharkie_123@db:5432/sharkie
CURRENCY_API_URL=https://currencyapi.net/api/v2/
CURRENCY_API_KEY=your_api_key_here
PORT=3000
ALLOWED_ORIGINS=
RUST_LOG=sharkie=info,tower_http=info
```

Note: `DATABASE_URL` uses `@db` — the Docker Compose service hostname.

### `sharkie_db.env` — PostgreSQL credentials

Used by the `db` service in `docker-compose.yml` via `env_file:`. Credentials must match the user/password/db in `DATABASE_URL`.

```bash
# sharkie_db.env.example — copy to sharkie_db.env

POSTGRES_USER=sharkie
POSTGRES_PASSWORD=sharkie_123
POSTGRES_DB=sharkie
```

### `sharkie.local.env` — Local dev overrides (gitignored)

Loaded first by `cargo run`. Override any vars that differ between Docker and local — typically `DATABASE_URL` (localhost vs Docker hostname) and log levels.

```bash
# sharkie.local.env.example — copy to sharkie.local.env

DATABASE_URL=postgresql://sharkie:sharkie_123@localhost:5432/sharkie
ALLOWED_ORIGINS=http://localhost:3001
RUST_LOG=sharkie=debug,tower_http=debug
```

Only vars that differ from `sharkie.env` need to be here. Everything else falls through.

### Variable reference

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `DATABASE_URL` | **yes** | — | PostgreSQL connection string |
| `CURRENCY_API_URL` | no | `https://api.currencyapi.com/v3/` | External currency rate API |
| `CURRENCY_API_KEY` | no | `""` | API key for rate fetching |
| `PORT` | no | `3000` | Server listen port |
| `ALLOWED_ORIGINS` | no | `*` (any) | Comma-separated CORS origins |
| `RUST_LOG` | no | `sharkie=info,tower_http=info` | Tracing log level filter |

### Frontend environment

The frontend uses a single build-time variable:

| Variable | Context | Value |
|----------|---------|-------|
| `NEXT_PUBLIC_API_BASE` | Local dev (default) | `https://sharkie.uha.app` |
| `NEXT_PUBLIC_API_BASE` | Docker build arg | `/api/proxy` |

To point the frontend at your local backend:

```bash
NEXT_PUBLIC_API_BASE=http://localhost:3000 pnpm dev
```

## Database Dumps

Database dumps are stored in `./dumps/` as PostgreSQL custom format (`.dump`) files.

### Auto-restore on `make dev`

When you run `make dev`, the restore script automatically:
1. Finds the latest `.dump` file in `./dumps/`
2. Checks if the database already has data
3. Restores the dump only if the database is empty
4. Skips silently if data already exists

### Manual restore

```bash
# Restore latest dump (skip if data exists)
make restore

# Force restore (truncate + restore)
make restore-force
```

### Download dumps from remote

```bash
make backup
```

Requires `BACKUP_REMOTE_DEFAULT_USER` and `BACKUP_REMOTE_DEFAULT_HOST` in `.env`.

## Code Quality

### Biome (Frontend)

The frontend uses [Biome](https://biomejs.dev/) for linting and formatting, replacing ESLint and Prettier:

```bash
cd frontend && pnpm lint       # Check for issues
cd frontend && pnpm fix        # Auto-fix issues
cd frontend && pnpm format     # Format files
cd frontend && pnpm typecheck  # Type check
```

### Rust

```bash
cargo fmt                      # Format
cargo clippy -- -D warnings    # Lint
cargo check                    # Check
```

### Combined

```bash
make check    # Run everything at once
```

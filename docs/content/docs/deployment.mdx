# Deployment

Sharkie is deployed as Docker containers orchestrated by Docker Compose, with images pushed to a private registry.

## Docker Architecture

### Backend (`Dockerfile`)

Multi-stage build:
1. **Build stage** ‚Äî `rust:1.93-bookworm` with system libs, compiles release binary
2. **Runtime stage** ‚Äî `debian:bookworm-slim` with minimal dependencies

### Frontend (`frontend/Dockerfile`)

Three-stage build:
1. **Dependencies** ‚Äî Node 22 Alpine with pnpm, installs packages
2. **Build** ‚Äî Runs `next build`, produces static export in `out/`
3. **Runtime** ‚Äî Nginx Alpine serving static files

### Docker Compose Services

| Service | Image | Port | Health Check |
|---------|-------|------|-------------|
| `db` | `postgres:17-alpine` | 5432 (internal) | `pg_isready` |
| `app` | `registry.keireira.com/sharkie:latest` | 4444 ‚Üí 3000 | `curl /health` |
| `frontend` | `registry.keireira.com/sharkie-fe:latest` | 3333 ‚Üí 3000 | `nginx -t` |

All services are on the `sharkie-network` bridge network.

## CI/CD Pipeline

Two GitHub Actions workflows handle the full pipeline:

### CI (`ci.yml`)

Runs on every push and PR to `master`. Two parallel jobs:

- **Backend checks** ‚Äî `cargo check`, `cargo clippy -D warnings`, `cargo-deny` advisory audit. Uses `Swatinem/rust-cache` for cargo caching.
- **Frontend checks** ‚Äî Biome lint, TypeScript type check, `pnpm audit`. Uses `actions/setup-node` with pnpm store caching.

### Deploy (`deploy.yml`)

Runs on push to `master` (excludes `docs/`, `rules/`, `*.md`). Uses `dorny/paths-filter` to detect which components changed and only rebuilds what's needed.

**Build Backend** (conditional on `src/`, `migrations/`, `Cargo.*`, `Dockerfile` changes):
- Docker Buildx with `docker/build-push-action`
- Layer caching via GitHub Actions cache (`type=gha, mode=max`)
- Pushes to `registry.keireira.com/sharkie:latest`

**Build Frontend** (conditional on `frontend/` changes):
- Docker Buildx with `docker/build-push-action`
- Layer caching via GitHub Actions cache (`type=gha, mode=max`)
- Pushes to `registry.keireira.com/sharkie-fe:latest`

**Deploy** (runs after both builds, skipped if nothing changed):
- Copies `docker-compose.yml` to server via SCP
- SSH: pulls images ‚Üí `docker compose up -d` ‚Üí prunes old images

## Manual Deployment

```bash
# Build and push images manually
docker buildx build --push -t registry.keireira.com/sharkie:latest .
docker buildx build --push -t registry.keireira.com/sharkie-fe:latest frontend

# Deploy on server
docker compose pull
docker compose up -d --remove-orphans
docker image prune -f
```

## Nginx Configuration

The frontend Nginx serves static files and proxies API requests:

```nginx
location /api/proxy/ {
    proxy_pass http://SHARKIE-APP:3000/;
    # Forwards X-Real-IP, X-Forwarded-For, X-Forwarded-Proto
}

location / {
    try_files $uri $uri.html $uri/index.html /index.html;
}
```

## Database Backups

A backup script downloads `.dump` files from a remote server:

```bash
make backup
# or
./download-backups.sh
```

Requires `BACKUP_REMOTE_DEFAULT_USER` and `BACKUP_REMOTE_DEFAULT_HOST` environment variables.

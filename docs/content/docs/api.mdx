# API Reference

The Sharkie API provides exchange rate data through several endpoints. All responses are JSON with gzip/brotli compression. An OpenAPI 3.1 spec is available at `/openapi.json`.

## Base URL

```
Production: https://sharkie.uha.app
Local:      http://localhost:3000
```

## Common Response Headers

Every response includes:

| Header | Value |
|--------|-------|
| `x-request-id` | UUID v4 (for debugging/correlation) |
| `x-ratelimit-limit` | `60` |
| `x-ratelimit-remaining` | Remaining requests in window |
| `strict-transport-security` | HSTS with preload |
| `x-content-type-options` | `nosniff` |
| `x-frame-options` | `DENY` |
| `referrer-policy` | `strict-origin-when-cross-origin` |
| `content-encoding` | `gzip` or `br` (if client supports) |

## GET /history

Returns historical exchange rates. Supports date ranges, specific dates, currency filtering, and base currency conversion.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | conditional | Start date (`YYYY-MM-DD`). Required with `to`. |
| `to` | string | conditional | End date (`YYYY-MM-DD`). Required with `from`. |
| `date` | string | conditional | Comma-separated dates (`YYYY-MM-DD`). Use instead of `from`/`to`. |
| `currencies` | string | optional | Comma-separated currency codes to filter. |
| `base` | string | optional | Base currency (default: `USD`). |
| `limit` | integer | optional | Max days to return, 1–366 (default: `366`). |
| `offset` | integer | optional | Number of days to skip for pagination (default: `0`). |

Either `from`+`to` or `date` must be provided. Date range maximum is 5 years.

### Timezone Handling

Dates in the future (relative to UTC) are automatically clamped to today. This prevents 404 errors when clients in timezones ahead of UTC (e.g., GMT+10) request "today's" rates that haven't been stored yet on the server.

### Cache Policy

| Condition | Cache-Control |
|-----------|--------------|
| All dates in the past | `public, s-maxage=86400, max-age=86400, immutable` |
| Includes today or future | `public, s-maxage=900, max-age=300` |

### Examples

**Date range:**
```
GET /history?from=2024-01-01&to=2024-01-31&currencies=EUR,GBP&base=USD
```

**Specific dates:**
```
GET /history?date=2024-01-01,2024-06-01,2024-12-01&currencies=EUR
```

**Non-USD base:**
```
GET /history?from=2024-01-01&to=2024-01-07&base=EUR&currencies=USD,GBP,JPY
```

**Paginated:**
```
GET /history?from=2024-01-01&to=2024-12-31&limit=100&offset=0
```

### Response

```json
{
  "base": "USD",
  "total": 365,
  "data": [
    {
      "date": "2024-01-01",
      "rates": {
        "EUR": 0.9013,
        "GBP": 0.7854
      }
    }
  ]
}
```

### Error Responses

| Status | Condition |
|--------|-----------|
| 400 | Invalid date format, missing parameters, empty currency list, date range exceeds 5 years |
| 404 | No data found for the given query |
| 408 | Request timed out (30s limit) |
| 413 | Response exceeds 1 MB limit |
| 429 | Rate limit exceeded (60 req/min) |
| 500 | Internal server error |

```json
{
  "status": "error",
  "code": 400,
  "message": "Invalid 'from' date: 'abc'. Use YYYY-MM-DD"
}
```

## Rate Limits

60 requests per minute per client IP. Tracked via `X-Forwarded-For` (Cloudflare/Nginx) with peer IP fallback. Response headers:

```
x-ratelimit-limit: 60
x-ratelimit-remaining: 42
retry-after: 1          # (only on 429)
```

## Response Size

Responses are capped at 1 MB. If a query would produce a larger response, a 413 error is returned. Use the `currencies` parameter to limit the data set.

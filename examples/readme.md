# Currency Formatting Properties

## `minor_unit` vs `decimal_places`

| Свойство | Назначение |
|----------|------------|
| `minor_unit` | **Реальная точность валюты** — сколько знаков после запятой существует в денежной системе страны |
| `decimal_places` | **Точность отображения** — сколько знаков показывать пользователю |

### Когда они различаются?

Для большинства валют (например, KZT) они совпадают (оба = 2), но бывают случаи:

- **Кувейтский динар (KWD)**: `minor_unit: 3` — реально существуют 1/1000 динара (филсы)
- **Японская иена (JPY)**: `minor_unit: 0` — нет дробных частей вообще
- **Биткоин**: `minor_unit: 8` — но `decimal_places` может быть 2 для удобства отображения

### Пример различия

```
Сумма: 123.456789 BTC
minor_unit: 8     → хранить/считать с точностью до 8 знаков
decimal_places: 2 → показывать как "₿123.46"
```

## `rounding` — правила округления

Это **шаг округления** для финальных сумм. Значение `0` означает «без специального округления».

Примеры где `rounding ≠ 0`:

| Валюта | rounding | Эффект |
|--------|----------|--------|
| CHF (швейцарский франк) | `5` | Округление до 0.05 (5 раппенов) — в Швейцарии нет монет 1 и 2 раппена |
| SEK (шведская крона) | `50` | Округление до 0.50 — Швеция отказалась от мелких монет |

Пример с CHF (`rounding: 5`):

```
1.02 → 1.00
1.03 → 1.05
1.07 → 1.05
1.08 → 1.10
```

## Итого

- `minor_unit` — для вычислений
- `decimal_places` — для UI
- `rounding` — для кассовых операций с наличными

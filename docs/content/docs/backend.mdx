# Backend

The backend is a Rust API server built with Axum 0.8 and SQLx 0.8, running on Tokio.

## Module Structure

```
src/
├── main.rs          # Entry point: bootstrap, middleware, graceful shutdown
├── config.rs        # Config struct loaded from environment variables
├── error.rs         # Unified ApiError enum with IntoResponse
├── state.rs         # AppState: DB pool, config, HTTP client
├── models.rs        # DB models (ExchangeRate) and API DTOs
├── routes/
│   ├── mod.rs       # Router assembly
│   ├── health.rs    # GET /health
│   ├── currencies.rs # GET /currencies
│   └── history.rs   # GET /history (date parsing, filtering, rebasing)
└── services/
    ├── mod.rs
    └── rates.rs     # Backfill, daily fetch, scheduler, batch insert
```

## Architecture Pattern

The codebase follows the **routes → services → db** pattern from the project rules:

1. **Routes** handle HTTP concerns (parsing query params, returning JSON)
2. **Services** contain business logic (backfill, scheduling, rate calculations)
3. **Database** access is done through SQLx with parameterized queries (SQL injection safe)

## Configuration

All configuration is via environment variables:

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `DATABASE_URL` | yes | — | PostgreSQL connection string |
| `CURRENCY_API_URL` | no | `https://api.currencyapi.com/v3/` | External rate API base URL |
| `CURRENCY_API_KEY` | no | `""` | API key for rate fetching |
| `PORT` | no | `3000` | Server listen port |
| `ALLOWED_ORIGINS` | no | `*` (any) | Comma-separated CORS origins |
| `RUST_LOG` | no | `sharkie=info,tower_http=info` | Log level filter |

## Error Handling

A unified `ApiError` enum covers all error cases:

- `NotFound` → 404
- `InvalidInput(String)` → 400
- `PayloadTooLarge` → 413
- `Internal(String)` → 500 (message logged, generic response to client)

Automatic `From` implementations for `sqlx::Error` and `reqwest::Error` allow using `?` in handlers.

## Background Tasks

### Backfill (`backfill_missing_rates`)
Runs on startup. Iterates from 2000-01-01 to yesterday and fetches any missing dates from the external API.

### Daily Updater (`daily_rate_updater`)
Background Tokio task that:
1. Checks if today's rates exist on startup; fetches if missing
2. Sleeps until the next scheduled UTC time
3. Schedule: 00:05, 04:00, 08:00, 12:00, 16:00, 20:00, 23:55

### Batch Insert
Both backfill and daily fetch use a single parameterized `INSERT ... ON CONFLICT DO UPDATE` query for efficiency.

## Middleware

- **CORS** — via `tower-http::cors`, configurable via `ALLOWED_ORIGINS`
- **Tracing** — structured logging via `tracing` + `tracing-subscriber` with env filter
- **Graceful Shutdown** — handles SIGTERM and Ctrl+C cleanly

## Database

Single table with composite primary key:

```sql
CREATE TABLE exchange_rates (
    currency_code VARCHAR(5) NOT NULL,
    date DATE NOT NULL,
    rate NUMERIC(20, 10) NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    PRIMARY KEY (currency_code, date)
);

CREATE INDEX idx_rates_date ON exchange_rates(date);
```

Migrations are managed by SQLx and run automatically on startup.

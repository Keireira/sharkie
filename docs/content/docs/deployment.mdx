# Deployment

Sharkie is deployed as Docker containers orchestrated by Docker Compose, with images pushed to a private registry.

## Docker Architecture

### Backend (`Dockerfile`)

Multi-stage build:
1. **Build stage** — `rust:1.93-bookworm` with system libs, compiles release binary
2. **Runtime stage** — `debian:bookworm-slim` with minimal dependencies

### Frontend (`frontend/Dockerfile`)

Three-stage build:
1. **Dependencies** — Node 22 Alpine with pnpm, installs packages
2. **Build** — Runs `next build`, produces static export in `out/`
3. **Runtime** — Nginx Alpine serving static files

### Docker Compose Services

| Service | Image | Port | Health Check |
|---------|-------|------|-------------|
| `db` | `postgres:17-alpine` | 5432 (internal) | `pg_isready` |
| `app` | `registry.keireira.com/sharkie:latest` | 4444 → 3000 | `curl /health` |
| `frontend` | `registry.keireira.com/sharkie-fe:latest` | 3333 → 3000 | `nginx -t` |

All services are on the `sharkie-network` bridge network.

## CI/CD Pipeline

Three GitHub Actions workflows:

### 1. Build Backend Image
- **Trigger:** Push to `master` (excluding `frontend/` paths)
- **Steps:** Checkout → Registry login → Docker build → Push → Cleanup

### 2. Build Frontend Image
- **Trigger:** Push to `master` (only `frontend/` paths)
- **Steps:** Checkout → Registry login → Docker build → Push → Cleanup

### 3. Deploy
- **Trigger:** After both image builds complete successfully
- **Steps:** SSH to server → Copy `docker-compose.yml` → Pull images → `docker compose up -d` → Prune old images

## Manual Deployment

```bash
# Build and push images manually
docker build -t registry.keireira.com/sharkie:latest .
docker push registry.keireira.com/sharkie:latest

docker build -t registry.keireira.com/sharkie-fe:latest frontend
docker push registry.keireira.com/sharkie-fe:latest

# Deploy on server
docker compose pull
docker compose up -d --remove-orphans
docker image prune -f
```

## Nginx Configuration

The frontend Nginx serves static files and proxies API requests:

```nginx
location /api/proxy/ {
    proxy_pass http://SHARKIE-APP:3000/;
    # Forwards X-Real-IP, X-Forwarded-For, X-Forwarded-Proto
}

location / {
    try_files $uri $uri.html $uri/index.html /index.html;
}
```

## Database Backups

A backup script downloads `.dump` files from a remote server:

```bash
make backup
# or
./download-backups.sh
```

Requires `BACKUP_REMOTE_DEFAULT_USER` and `BACKUP_REMOTE_DEFAULT_HOST` environment variables.

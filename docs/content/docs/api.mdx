# API Reference

The Sharkie API provides exchange rate data through three endpoints. All responses are JSON.

## Base URL

```
Production: https://sharkie.uha.app
Local:      http://localhost:3000
```

---

## GET /health

Returns server and database status. Returns `200 OK` when healthy, `503 Service Unavailable` when the database is unreachable.

**Response (healthy):**
```json
{
  "status": "ok",
  "message": "Sharkie is running",
  "db": true
}
```

**Response (degraded):**
```json
{
  "status": "degraded",
  "message": "Sharkie is running",
  "db": false
}
```

---

## GET /currencies

Returns all available currency codes.

**Response:**
```json
{
  "currencies": ["AED", "ARS", "AUD", "BDT", "BHD", "..."]
}
```

## GET /history

Returns historical exchange rates. Supports date ranges, specific dates, currency filtering, and base currency conversion.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | conditional | Start date (`YYYY-MM-DD`). Required with `to`. |
| `to` | string | conditional | End date (`YYYY-MM-DD`). Required with `from`. |
| `date` | string | conditional | Comma-separated dates (`YYYY-MM-DD`). Use instead of `from`/`to`. |
| `currencies` | string | optional | Comma-separated currency codes to filter. |
| `base` | string | optional | Base currency (default: `USD`). |

Either `from`+`to` or `date` must be provided.

### Examples

**Date range:**
```
GET /history?from=2024-01-01&to=2024-01-31&currencies=EUR,GBP&base=USD
```

**Specific dates:**
```
GET /history?date=2024-01-01,2024-06-01,2024-12-01&currencies=EUR
```

**Non-USD base:**
```
GET /history?from=2024-01-01&to=2024-01-07&base=EUR&currencies=USD,GBP,JPY
```

### Response

```json
{
  "base": "USD",
  "data": [
    {
      "date": "2024-01-01",
      "rates": {
        "EUR": 0.9013,
        "GBP": 0.7854
      }
    },
    {
      "date": "2024-01-02",
      "rates": {
        "EUR": 0.9021,
        "GBP": 0.7862
      }
    }
  ]
}
```

### Error Responses

| Status | Condition |
|--------|-----------|
| 400 | Invalid date format, missing parameters, empty currency list, date range exceeds 5 years |
| 404 | No data found for the given query |
| 413 | Response exceeds 512 KB limit |
| 500 | Internal server error |

```json
{
  "status": "error",
  "code": 400,
  "message": "Invalid 'from' date: 'abc'. Use YYYY-MM-DD"
}
```

## Rate Limits

Currently no rate limiting is enforced at the API level. Rate limiting should be implemented at the reverse proxy layer for production use.

## Response Size

Responses are capped at 512 KB. If a query would produce a larger response, a 413 error is returned. Use the `currencies` parameter to limit the data set.
